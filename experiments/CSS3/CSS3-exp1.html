Documentation

1. CSS3 animations are created using the @keyframes rule.

2. Here is the @keyframes rule that I use to produce the glow effect.

from{
      text-shadow: 0 0 10px #fff, 0 0 20px #fff, 0 0 30px #fff, 0 0 40px #FF1177, 0 0 70px #FF1177, 0 0 80px #FF1177, 0 0 100px #FF1177, 0 0 150px #FF1177;
    }
to {
      text-shadow: 0 0 5px #fff, 0 0 10px #fff, 0 0 15px #fff, 0 0 20px #FF1177, 0 0 35px #FF1177, 0 0 40px #FF1177, 0 0 50px #FF1177, 0 0 75px #FF1177;
   }

Explanation of @keyframe rule used in this experiment:

1. In this example, I have used the text-shadow property of CSS3 to produce the animation effect

2. Attributes of text-shadow that I used: 

text-shadow [h-shadow] [v-shadow] [blur] [color]

3. To produce the text-shadow effect as an animation , I am passing different arguments to the [blur] and [color] parameters of text-shadow

4. 'from' attribute of @keyframe holds the list of values that it takes over the animation cycle.

5. 'to' attribute holds the list of values that the @keyframes rule takes when the animation completes a cyle.

Explanation of Animation function used in this experiment:

1. Animation cycle is the period of time over which the animation is configured to occur. Here is the configuration of my animation function

            -webkit-animation-name: Arul_glow_function;
            -webkit-animation-duration: 1.5s; 
            -webkit-animation-direction:alternate;
            -webkit-animation-timing-function: ease-in-out;
            -webkit-animation-direction: alternate;
            -webkit-animation-iteration-count:infinite;
/*The parameters in the animation function are self-explanatory*/

2. However, the word 'webkit' demands a mention. Webkit is a layout engine software component used for rendering web pages in web browsers like Chrome and Safari.

3.Different browsers use different layout engines. For example Mozilla uses '-moz' . For your animation to work in all the browsers that you want them to work , you will have to configure their corresponding layout engine , just as I have done for webkit

Wrapping Up

1.We aren't done yet. Here's the final and important step. We will have to bind our animation function with the component that we want to animate. I wanted the animation to affects to be produced when the user hovers over a text. Here is the code 

.p_glow_text1:hover 
{
            -moz-animation: Arul_glow_function 1.5s ease-in-out infinite alternate;
}
}

2. I have also used the CSS3 transition function to let the animations transition from one style to another smoothly
