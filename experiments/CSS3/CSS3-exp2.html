Purpose of the Experiment: To understand Pseudo-elements & pseudo-classes and create a utility that serves as an illustration of how they can be put together to create niche styles

What does the experiment do? It displays the URL of a link , when the user hovers over a link

Illustration of the Experiment:

1.In this experiment, I attempt to style an Pseudo-element in response to a Pseudo class .

2.Here is the code that generates the Pseudo element from the real element-'a'

   a:before {
                -webkit-transform: translate3d(0, 0, 0);
                content: attr(href);
                font-size: 15px;
                position: absolute;
                width: 0%;
                height: 50%;
                top: 0;
                right: 0;
                border-bottom: 6px solid #FFF;
                z-index: -1;
                transition: all .4s ease;
                opacity: 0;
                overflow: hidden;
            }

3.Pseudo Element created here is the href attribute of the element 'a'.

4.Pseudo Element is placed before the element 'a' as dictated by the pseudo-element keyword ':before'

5.Oh wait, where did the pseudo-element go, if it was placed before the element 'a'? .
We make the pseudo element invisible by stacking it behind the real element 'a' , by settings its opacity property to 1 ,by setting its width to 0, by setting its top and right positions as zero.You can alter these properties to find the pseudo-element lying alongside the real 'a' element.

6.When the user hovers over the link , we bring forward the pseudo-element by altering its properties that would make the pseudo-element visible to the user.

a:hover:before {
                width: 100%;
                left: 0;
                opacity: 1;
                top: -50%;
               }
